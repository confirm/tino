❤️ Support
----------

TINO is a community-driven open-source project. We do our best to help, but please understand that support is provided on a voluntary basis.

Getting help
~~~~~~~~~~~~

- **GitHub Issues** — for bug reports and feature requests, please open an issue on the `TINO GitHub repository <https://github.com/confirm-solutions/tino>`_.
- **Discussions** — for questions, ideas, and general conversation, use `GitHub Discussions <https://github.com/confirm-solutions/tino/discussions>`_.

Before opening an issue, please:

1. Check the documentation — your question may already be answered here.
2. Search existing issues to avoid duplicates.
3. Include your TINO version, deployment method, and steps to reproduce the problem.

Typst support
~~~~~~~~~~~~~

For questions about the Typst language itself — syntax, packages, or the compiler — please refer to the `Typst documentation <https://typst.app/docs/>`_ and the `Typst community <https://typst.app/community/>`_. The Typst team maintains its own support channels and we encourage you to use them.

If you are evaluating whether TINO or Typst's own editor is the right fit, see the :ref:`Introduction` for how they compare.

Commercial support
~~~~~~~~~~~~~~~~~~

TINO is developed by `confirm IT <https://confirm.ch/>`_. Organisations that need commercial support or a commercial license without copyleft obligations can `contact us <https://confirm.ch/en/#contact>`_.

Also consider `Typst's commercial editor <https://typst.app/>`_, which can be self-hosted and includes dedicated support from the makers of the language itself (see important message on :doc:`index <index>`).
